## Core Concepts of Self Attention
The self attention mechanism is a fundamental component of transformer models, allowing them to weigh the importance of different input elements relative to each other. To understand self attention, we must first derive its equation from first principles. The self attention equation is derived by considering the input sequence as a set of vectors, where each vector represents a token or element in the sequence. The self attention mechanism computes the weighted sum of these vectors, where the weights are learned based on the input vectors themselves.

* Derive the self attention equation from first principles:
  The self attention equation can be derived as follows: 
  ```python
  # Self attention equation
  Attention(Q, K, V) = softmax(Q * K^T / sqrt(d)) * V
  ```
  where `Q`, `K`, and `V` are the query, key, and value vectors, respectively, and `d` is the dimensionality of the input vectors.

* Compare self attention with traditional attention mechanisms:
  Self attention differs from traditional attention mechanisms in that it does not rely on recurrent neural networks (RNNs) or convolutional neural networks (CNNs) to compute the attention weights. Instead, self attention uses the input vectors themselves to compute the attention weights, allowing for parallelization and improved performance.

* Explain the role of query, key, and value vectors in self attention:
  In self attention, the query, key, and value vectors play the following roles:
  + Query vector (`Q`): represents the context in which the attention is being computed
  + Key vector (`K`): represents the input elements being attended to
  + Value vector (`V`): represents the values being weighted and summed
  The query, key, and value vectors are typically obtained by linearly transforming the input vectors, allowing the self attention mechanism to capture complex relationships between the input elements. By using these vectors, self attention can effectively weigh the importance of different input elements and compute a weighted sum of the value vectors.

## Implementing Self Attention
To implement self attention in a deep learning model, we can utilize the TensorFlow library. 
A key component of self attention is the ability to compute attention weights based on input data.
```python
import tensorflow as tf

# Define self attention layer
class SelfAttention(tf.keras.layers.Layer):
    def __init__(self, embed_dim, num_heads):
        super(SelfAttention, self).__init__()
        self.embed_dim = embed_dim
        self.num_heads = num_heads
        self.query_dense = tf.keras.layers.Dense(embed_dim)
        self.key_dense = tf.keras.layers.Dense(embed_dim)
        self.value_dense = tf.keras.layers.Dense(embed_dim)
        self.combine_heads = tf.keras.layers.Dense(embed_dim)

    def call(self, x):
        # Compute query, key, and value vectors
        query = self.query_dense(x)
        key = self.key_dense(x)
        value = self.value_dense(x)
        # Compute attention weights
        attention = tf.matmul(query, key, transpose_b=True)
        attention = tf.nn.softmax(attention)
        # Compute output
        output = tf.matmul(attention, value)
        output = self.combine_heads(output)
        return output
```

## Common Mistakes in Self Attention
Self attention is a powerful mechanism, but its implementation can be tricky. 
* Vanishing gradients can occur when the input sequence is long, causing the gradients to become smaller as they backpropagate through the network, which can lead to slower training or convergence to a suboptimal solution. 
To mitigate this, techniques like gradient clipping or using pre-trained models can be employed.

When handling edge cases, such as zero-length input sequences, a simple solution is to pad the input with a special token or return a default value.

This involves masking certain positions in the input sequence to prevent the model from attending to them, which can significantly reduce the number of computations required.

The trade-offs between self attention and recurrent neural networks (RNNs) are also important to consider.

However, for very long sequences, RNNs may be a better choice due to their ability to handle sequential dependencies more efficiently.
